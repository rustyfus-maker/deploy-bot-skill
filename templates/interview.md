# Deploy-Bot Interview Template

Ask these only after inspecting the codebase.

1. Preferred cloud: AWS, Azure, GCP, hybrid, cheapest, easiest, or choose for me?
2. Environment: demo, staging, production, enterprise production?
3. Traffic: expected users/requests now and in 12 months?
4. Data: relational DB, document DB, file uploads, cache, queue, cron, websockets?
5. Secrets: which third-party APIs and credentials are needed?
6. Compliance: region, retention, private networking, audit, encryption, regulated data?
7. Runtime style: serverless, containers, Kubernetes, VM, or choose for me?
8. Package format: Terraform, CDK, Bicep, Pulumi, Kubernetes, GitHub Actions, Docker?
